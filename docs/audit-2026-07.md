# Full adversarial audit — July 2026

A 13-agent adversarial audit swept every subsystem for one failure class: **claiming or
implying review that didn't happen** (silent omission, vacuous pass, buried severity,
N/A abuse, fabricated verdict basis) plus general correctness. 9 auditors completed;
**4 slices did not run** (session limits): schemas/CLI surfaces, corpus/eval internals,
docs-vs-reality, and the cross-cutting synthesis. Those four slices are still owed a pass.

72 raw findings. Triage rule: verdict-integrity and gate-integrity fixes land now;
anything touching the *validated grading rubric* is batched for the next paid
revalidation (changing rubric text invalidates the measured false-green numbers);
anything needing new fixtures waits for the fixtures (a critical without ≥4 diverse bad
fixtures would be an unvalidated number — the exact thing this tool refuses to ship).

## ⚠ Standing consequence

Several fixes touch the grading path (adapter assertion presentation, fail-closed verdict
validation, abuse-controls selection keywords, baas storage reasoning). The published
validation numbers (ecommerce ≤2.9%, newer archetypes ≤4.5%) were measured **before**
these changes. An impact analysis (see VALIDATION-NOTES "Revalidation owed") then PROVED the stale
scope is narrow: only `baas.rls_enforced`'s ~8 trials are invalidated — no other critical's
prompt or the corpus-eval path changed. The owed paid re-measurement is ~10–16 calls.

## Fixed in this pass (by finding)

**Verdict plumbing (verify.mjs / report-html.mjs)**
- Inconclusive runs: gate requires assessed evidence; terminal, JSON (`rollup.conclusive`),
  and HTML (Inconclusive badge) all say NOT a pass — no fabricated "criticals below the line".
- One shared rollup object across text/JSON/HTML — surfaces can no longer disagree.
- HTML states the real gate tier (was hardcoded "critical") and real severities (was
  labeling every level-3 "high-severity").
- Gate tiers come from checkpoint *definitions* (or the manifest's declared
  `goal_definition.gate_tiers`), never from what happened to be assessed; an all-N/A gate
  tier demotes **loudly** on every surface instead of silently.
- Portfolio declares `gate_tiers: ["critical","high"]` — its design/web bar gates again
  (adding criticals had silently pushed the highs out of the gate).
- Whole-domain omission (design skipped by the default `--domain backbone`) disclosed in
  text, JSON (`rollup.design_skipped`), and HTML.
- Mock-fallback degradation warned on stderr in every mode, carried as
  `rollup.adapter_degraded`, and stamped on the terminal + HTML verdict.
- `--domain` validated; invalid severity overrides refused with a warning; challenged vs
  structural N/A get distinct wording.

**PR gate (pr-gate.mjs / action.yml)**
- A failed git diff is an error (exit 2), never an empty diff (which green-lit merges).
- Fallback-mock cannot certify a merge: warned, marked, and **fails closed** under `--fail`.
- A PR that edits the gate's own rules (forespec.config.json / .forespec/) is flagged in
  the comment and fails closed under `--fail`. (Full fix — reading rules from the base
  ref — tracked below.)
- Changed files that exist but can't be scanned (>64KB, unsupported) are named.
- Regressions block at ANY severity (a high slipping 9→3 was invisible); "pass" wording
  no longer claims a tier was checked when nothing in it was touched.
- Model N/A verdicts handled (were rendered as errors); ungraded checkpoints are named in
  the needs-a-look comment; comment-post failures are detected and the report printed.
- Priors scoped by archetype+adapter (a shared store could cross-contaminate regression
  math); passive flagged→fixed outcomes join the REAL prior prediction (was a synthetic
  chimera record); `all-domains` input added to the GitHub Action.

**Adapter (claude.mjs)**
- Fail closed on malformed verdicts: level must be 3|6|9; a passing level with no
  rationale (no stated basis) is an error; confidence clamped to [0,1].
- Runtime-typed assertions no longer presented as required-in-code properties for a 6.
- 408/504 added to the retryable set.

**Detection / plan / start**
- "store" demoted from strong intent tells ("a tool to store recipes" now asks).
- `<<<…>>>` marker forgery stripped from untrusted self-description text.
- AI classifier failures print a note instead of being indistinguishable from "no result";
  low-confidence AI picks no longer auto-proceed.
- `plan` never silently defaults to ecommerce — it asks; `--checkpoint <id>` always plans
  the named checkpoint; every plan discloses what it does NOT cover (count + omitted
  criticals by id); `--domain` validated; `--store` parsed correctly; `--json --out` writes.

**Selection**
- The budget packer clips a top-ranked file into the remaining budget instead of skipping
  it while lower-ranked noise back-fills (the Juice-Shop-class false-green path).
- `security.abuse_controls` gets subject keywords (login/signin/signup/password/otp) — a
  repo with NO abuse controls anywhere no longer reads structurally N/A.

**Archetypes / library / store**
- ai-app v1.2.0: `auth.access_control` → critical, `auth.credential_storage` added critical
  (both have ≥5 bad fixtures), `auth.session_security` + `security.file_upload` added high.
- ecommerce v2.1.0: `auth.session_security` added high.
- `baas.rls_enforced` reasoning covers storage buckets/rules (additive).
- resolve.mjs errors on a missing/empty `checkpoints` array (a typo'd key resolved to a
  clean-looking empty archetype).
- Self-test: universal-critical guard now fails on omission, not just downgrade; bandFor
  test probes the real 0.4/0.75 boundaries; corrupt store lines skipped with a warning.
